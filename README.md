## 🎯 About the Project

**VidCognify** is a full-stack, AI-powered YouTube intelligence platform built for students, researchers, and lifelong learners who want to extract maximum knowledge from any YouTube video — without watching the entire thing.

Instead of spending hours watching videos, **VidCognify gives you AI-generated summaries, chapter breakdowns, quizzes, study notes, video comparisons, and curated learning paths** — all in one seamless dashboard.

> *"YouTube has millions of educational videos. But watching all of them is impossible. VidCognify makes every minute count."*

## 🗄️ Database Schema

Your database is hosted on **PostgreSQL** and managed via **Prisma ORM**:

### Authentication & User Handling
| Model | Purpose |
|---|---|
| `User` | Core user profile — name, email, avatar, preferences |
| `Session` | Active login sessions with expiry tracking |
| `Account` | OAuth account linking (future Google/GitHub login) |
| `Verification` | Email verification tokens |
| `SecurityQuestion` | Pre-defined security questions for password recovery |
| `UserSecurityQuestion` | User's chosen questions with bcrypt-hashed answers |

### Content & AI Deliverables
| Model | Purpose |
|---|---|
| `Summary` | Master table — video URL, title, thumbnail, transcript text, AI summary, category, difficulty, sentiment, language, keywords |
| `TranscriptSegment` | Granular subtitle segments with millisecond offsets and durations |
| `Topic` | AI-detected video chapters — title, startMs, endMs, linked to Summary |
| `Highlight` | User-saved text highlights with color tag (yellow/green/blue/pink) and optional notes |

### User Configuration
| Model | Purpose |
|---|---|
| `UserTopicEdit` | Stores user's manual overrides for AI topic titles and timestamps |
| `UserApiKey` | AES-encrypted storage of user-provided Groq/OpenAI/Anthropic API keys |
| `ApiUsageLog` | Token consumption tracking per summarization call for analytics |
| `AppConfig` | System-level environment configuration values |

## 🔌 API Routes

### Core Feature APIs
| Route | Method | Description |
|---|---|---|
| `/api/summarize` | POST | Main pipeline — fetch transcript via YouTube API, call AI for summary, chapters, sentiment, keywords, save to DB |
| `/api/chat` | POST | Context-aware chatbot — load video transcript from DB, stream AI response to user's question |
| `/api/compare` | POST | Fetch both video transcripts, call AI for side-by-side analysis, return winner + unique strengths |
| `/api/notes` | POST | Generate Markdown-formatted study notes from video transcript |
| `/api/quiz` | POST | Generate 5 MCQs from transcript with correct answers and explanations |
| `/api/learning-path` | GET | Fetch 5 curated YouTube videos via YouTube Data API based on summary keywords/category |
| `/api/highlights` | GET/POST/DELETE | CRUD operations for user text highlights |
| `/api/topics/edit` | PUT | Save user's manual topic title/timestamp overrides |

### Auth & Security APIs
| Route | Description |
|---|---|
| `/api/auth/[...all]` | Better-Auth core — login, signup, session management, token refresh |
| `/api/security-questions/setup` | Save user's chosen security question + hashed answer during onboarding |
| `/api/security-questions/verify` | Verify answer for password reset flow |
| `/api/security-questions/reset-password` | Reset password after security question verification |
| `/api/account/delete` | Permanently delete user account and all associated data |

### Data & Settings APIs
| Route | Description |
|---|---|
| `/api/dashboard` | Fetch user analytics — total videos, streak, weekly activity, category breakdown |
| `/api/history` | Paginated list of user's past summaries |
| `/api/history/[id]` | Full details of a specific past summary |
| `/api/settings/api-keys` | Validate, AES-encrypt, and store user's custom AI API keys |
| `/api/setup/*` | Onboarding flow — completion status, validation, initial configuration |
| `/api/health` | System health check endpoint |

## 🧠 Key Technical Challenges Solved

> Real problems debugged and solved during development — not just theory.

| Challenge | Root Cause | Solution Applied |
|---|---|---|
| **Learning path showing kids/preschool videos** instead of topic-related content | `videoId` in URL had extra query params (`?detail=5&model=...`), DB lookup failed, fallback query `"educational tutorial"` was too generic | Cleaned videoId with `.split("?")[0]` before DB lookup. Improved fallback to use actual video keywords from DB, defaulting to `"technology tutorial"` instead |
| **Summary found: false** for every learning path request | VideoId mismatch between summary URL and DB stored ID — full URL was being stored vs. just the videoId | Fixed `findFirst` where clause to compare only the base videoId, not the full URL with params |
| **404 returned** when summary not found, blocking YouTube API call | Route short-circuited with 404 before ever calling YouTube Data API | Removed hard 404 return, implemented graceful fallback — always continue to YouTube API call regardless of DB hit/miss |
| **YouTube API blocked** on localhost | YouTube Data API v3 blocks requests from `localhost` origins | Configured API key restrictions in Google Cloud Console to allow local IPs during dev. Production on Vercel works without restriction |
| **Groq 429 Rate Limit** errors on free tier | Free tier allows only 2-3 API calls per minute. Concurrent feature use triggers limits | Added user-facing messaging to wait 1-2 minutes. Recommended BYOK for heavy users. Added graceful error handling instead of crashing |
| **API keys stored in plain text** | Initial implementation stored keys directly in DB | Migrated to AES encryption using `UserApiKey` model — keys encrypted before insert, decrypted only at point of use |
| **Dark mode flash** on page load | CSS variables not applied before React hydration | Used `next-themes` with `attribute="class"` and `suppressHydrationWarning` to prevent FOUC (Flash of Unstyled Content) |

## ⚙️ Installation & Setup

### Prerequisites
```bash
Node.js >= 18.x
npm >= 9.x
PostgreSQL database (Neon recommended — free tier available)
Groq API key (free at console.groq.com)
YouTube Data API v3 key (free at console.cloud.google.com)
```

### 1. Clone the Repository
```bash
git clone https://github.com/PrashantSinghUP64/VidCognify.git
cd VidCognify
```

### 2. Install Dependencies
```bash
npm install
```

### 3. Set Up Environment Variables

Create a `.env.local` file in the root directory:

```env
# Database
DATABASE_URL=postgresql://user:password@host/dbname

# Authentication (Better Auth)
BETTER_AUTH_SECRET=your_random_secret_key_here
BETTER_AUTH_URL=http://localhost:3000

# AI APIs
GROQ_API_KEY=gsk_your_groq_api_key
ANTHROPIC_API_KEY=sk-ant-your_anthropic_key   # optional
OPENAI_API_KEY=sk-your_openai_key              # optional

# YouTube Data API
YOUTUBE_API_KEY=AIza_your_youtube_api_key

# Encryption (for BYOK key storage)
ENCRYPTION_KEY=your_32_character_encryption_key

# Captcha (optional)
CAPTCHA_SECRET_KEY=your_captcha_secret
```

### 4. Set Up Database
```bash
# Generate Prisma client
npx prisma generate

# Push schema to database
npx prisma db push

# (Optional) Seed security questions
npx prisma db seed
```

### 5. Run Development Server
```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000) — you should see the VidCognify homepage!

### 6. Test the Core Feature
1. Register/Login
2. Paste any YouTube URL in the home input
3. Click Summarize
4. Wait 5–10 seconds
5. See AI-generated summary, chapters, sentiment, and more!
